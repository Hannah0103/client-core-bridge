## 本工程实现了客户端“Android、IOS双端共享同一套数据逻辑”，基于C++数据层实现数据隔离。

``本工程已实现 Android/iOS 双端通过 JNI/ObjC++ 薄桥接，共享同一份 C++ 数据核心输出（data-core），实现数据来源统一与逻辑隔离。
当前版本聚焦最小闭环（统一 payload + 双端消费）；缓存、diff、预取、平台能力注入等将作为下一阶段能力演进。``


### 一次实现，多端复用
- 数据加载、缓存、diff、预取、容错这些逻辑放在 C++，Android/iOS 都走同一套实现，避免多份代码长期漂移。

### 强一致性（尤其接口行为）
- 服务接口，双端通过各自桥接层接进去（JNI/ObjC++），能保证同一输入下行为基本一致，减少端间“同需求不同结果”。

### 把平台差异压缩到薄适配层
- 平台相关能力（日志、网络、文件、设备信息）通过注册注入；C++ 核心只依赖抽象接口。这样业务迭代主要改核心层，端上改动小。

### 历史包袱与工程现实
- iOS/Android 技术栈差异太大，如果直接三端各写一套数据层，维护成本和回归风险会持续上升。C++ 是跨端可控、性能和工具链都成熟的折中方案。
